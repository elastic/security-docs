Suspicious Activity Reported by Okta Useredit

This rule detects when a user reports suspicious activity for their Okta account. These events should be investigated, as they can help security teams identify when an adversary is attempting to gain access to their network.

Rule type: query

Rule indices:

  • filebeat-*

Severity: medium

Risk score: 47

Runs every: 5 minutes

Searches indices from: now-6m (Date Math format, see also Additional look-back time)

Maximum alerts per execution: 100

References:

  • https://developer.okta.com/docs/reference/api/system-log/
  • https://developer.okta.com/docs/reference/api/event-types/

Tags:

  • Elastic
  • Okta

Version: 1

Added (Elastic Stack release): 7.9.0

Rule authors: Elastic

Rule license: Elastic License

Potential false positivesedit

A user may report suspicious activity on their Okta account in error.

Rule queryedit

event.module:okta and event.dataset:okta.system and
event.action:user.account.report_suspicious_activity_by_enduser
